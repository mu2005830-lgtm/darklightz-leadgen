## 1. Scraper became slower

- Browser loading

- Page scrolling
- Extraction
- Database writes
- UI responsiveness

---

## 2. Scraper stops too early

This is currently the biggest issue.

If I request:

50 Leads

the scraper finds only a few businesses (sometimes only 3) and then stops.

This is incorrect.

The scraper should continue until:

- 50 unique qualifying leads have been collected

OR

- Google Maps has absolutely no more qualifying businesses available.

It must never stop after only a few results.

---

## 3. Replace skipped businesses

Whenever a business is skipped because:

- it has a website
- it is a duplicate
- it has missing required information
- validation fails

Immediately continue searching.

Skipped businesses should NEVER reduce the requested lead count.

Example:

Requested:

50 Leads

If 20 businesses are skipped,

continue searching until 50 valid businesses are collected.

---

## 4. Duplicate Detection

The application is saving duplicate businesses.

This must be fixed.

Before saving a business, compare against the ENTIRE database using:

- Business Name
- Phone Number
- Google Maps URL
- Coordinates (if available)

If it already exists anywhere in the database:

Skip it automatically.

Continue searching for another business.

Never save duplicates.
